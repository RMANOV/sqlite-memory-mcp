# v0.2.0 — Cross-Machine Bridge Sync via Git

**sqlite-memory-mcp now supports syncing knowledge graph entities between machines using a private git repo as transport.**

---

## What's new

### Bridge Tools (13-15)

Three new tools for cross-machine memory sync:

| Tool | Description |
|------|-------------|
| `bridge_push(tag="shared")` | Export entities where `project LIKE 'shared%'` to JSON, git commit + push |
| `bridge_pull()` | Git pull + import new entities/observations/relations with automatic dedup |
| `bridge_status()` | Compare local shared entities vs repo contents, show sync diff |

### How it works

```
┌─────────────────┐                          ┌─────────────────┐
│ Personal Machine │                          │  Work Machine    │
│                  │                          │                  │
│  memory.db       │   bridge_push()          │  memory.db       │
│  ┌────────────┐  │──────────┐               │  ┌────────────┐  │
│  │ entities   │  │          │               │  │ entities   │  │
│  │ project=   │  │    ┌─────▼─────────┐     │  │ project=   │  │
│  │ "shared:*" │  │    │ GitHub Private │     │  │ "shared:*" │  │
│  └────────────┘  │    │ Repo (bridge/) │     │  └────────────┘  │
│                  │    │               │     │                  │
│  bridge_pull()   │    │ shared.json   │     │  bridge_pull()   │
│  ◄───────────────│────┘               │─────│──────────────►   │
│                  │                    │     │                  │
└─────────────────┘    └───────────────┘     └─────────────────┘
```

1. Tag entities for sharing with `project="shared"` (or `"shared:topic"`)
2. `bridge_push()` exports to `shared.json` in a local git repo, commits, pushes
3. `bridge_pull()` on the other machine imports with dedup via UNIQUE constraints
4. Zero schema changes — uses the existing `project` field as a namespace

### Design decisions

- **Git as transport** — no cloud services, no API keys, works offline, audit trail via git log
- **UNIQUE constraints for dedup** — `INSERT OR IGNORE` means pulling the same entity twice is a no-op
- **Subprocess safety** — git commands use `capture_output=True` to never leak to stdout (would break MCP stdio)
- **Selective sync** — only entities with `project LIKE 'shared%'` are synced, everything else stays local

## Setup

```bash
# One-time on each machine
mkdir -p ~/.claude/memory/bridge
cd ~/.claude/memory/bridge
gh repo create memory-bridge --private --source=. --push

# Add BRIDGE_REPO env to settings.json
```

On second machine: `git clone` the private repo to `~/.claude/memory/bridge`.

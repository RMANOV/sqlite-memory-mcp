# v0.3.0 — Task Management, Kanban Board, Bridge Sync v2

**sqlite-memory-mcp now includes a full GTD-style task manager, a static Kanban HTML report, and extended cross-machine bridge sync with tasks.**

Tool count: 15 → 21.

---

## What's new

### Task Management — Tools 16-21

Six new MCP tools for task lifecycle management:

| Tool | Description |
|------|-------------|
| `create_task(title, ...)` | Create a task with section, priority, status, due date, parent, recurrence |
| `update_task(task_id, ...)` | Update any field on an existing task |
| `query_tasks(section?, priority?, status?, ...)` | Filter tasks by any combination of fields |
| `task_digest()` | GTD morning briefing — today's tasks, overdue, inbox count |
| `archive_done_tasks()` | Bulk-archive completed and cancelled tasks |
| `bump_overdue_priority()` | Escalate priority of tasks past their due date |

#### GTD Sections

```
inbox → today → next → someday → waiting
  │        │       │        │        │
  │   (do   │  (do  │  (maybe │  (blocked
  │  today) │  soon)│  later) │   on X)
  ▼        ▼       ▼        ▼        ▼
 triage  execute backlog  incubate  track
```

Tasks move through sections as they progress. `task_digest()` surfaces what matters now.

#### Schema

```sql
CREATE TABLE tasks (
    id           TEXT PRIMARY KEY,       -- UUID
    title        TEXT NOT NULL,
    description  TEXT,                   -- unlimited length
    section      TEXT NOT NULL DEFAULT 'inbox',
    priority     TEXT NOT NULL DEFAULT 'medium',
    status       TEXT NOT NULL DEFAULT 'not_started',
    due_date     TEXT,
    parent_id    TEXT REFERENCES tasks(id),  -- subtasks via self-ref FK
    recurrence   TEXT,                       -- JSON config
    created_at   TEXT NOT NULL,
    updated_at   TEXT NOT NULL
);
```

**Priorities:** `low` | `medium` | `high` | `critical`

**Statuses:** `not_started` | `in_progress` | `done` | `archived` | `cancelled`

**Subtasks** via `parent_id` self-referencing foreign key — unlimited nesting.

**Recurring tasks** via JSON config in the `recurrence` field (e.g. `{"interval": "weekly", "days": ["mon"]}`).

### Kanban Board — task_report.py

`task_report.py` generates a static HTML file with CSS Kanban columns from your tasks table:

- One column per section (Inbox / Today / Next / Someday / Waiting)
- Priority color coding (low=grey, medium=blue, high=orange, critical=red)
- Subtask count badges
- Overdue indicators
- Publishable via GitHub Pages — no server required

```bash
python3 task_report.py > board.html
# Open in browser or push to gh-pages branch
```

### Bridge Sync v2

Bridge payload extended to include tasks alongside entities. Last-write-wins for tasks, compared by `updated_at`. Backward compatible with v1 payloads (entities-only bridges continue to work).

`bridge_push()` and `bridge_pull()` now sync both entities and tasks in a single commit.

---

### B-tree Indexes

Added covering indexes on `entities`, `observations`, and `relations` for query performance under large graphs:

```sql
CREATE INDEX idx_entities_project  ON entities(project);
CREATE INDEX idx_entities_type     ON entities(entity_type);
CREATE INDEX idx_observations_eid  ON observations(entity_id);
CREATE INDEX idx_relations_from    ON relations(from_id);
CREATE INDEX idx_relations_to      ON relations(to_id);
```

---

### Bug fix — DB init on import

`_init_db()` was previously called only inside `if __name__ == "__main__"`. When `server.py` was imported as a library (e.g. for testing or embedding), tables were never created. Fixed: `_init_db()` now runs at module import time.
